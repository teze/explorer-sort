## 测试步骤

### 准备工作
1. 在 VS Code 中打开此项目
2. 确保已运行 `npm install` 和 `npm run compile`
3. 按 F5 启动扩展开发主机

### 基本功能测试
1. 在扩展开发主机中打开一个测试文件夹
2. 查看资源管理器工具栏是否显示排序图标
3. 点击图标，测试每种排序方式
4. 观察文件排序是否正确变化
5. 检查是否显示成功/警告消息

### 持久化测试
1. 选择一种排序方式（如"按类型排序"）
2. 检查 `.vscode/settings.json` 是否更新
3. 重新加载窗口（Ctrl+R / Cmd+R）
4. 确认排序方式保持不变

### 多工作区测试
1. 在工作区 A 选择"按名称排序"
2. 在工作区 B 选择"按修改时间排序"
3. 切换回工作区 A，确认仍是"按名称排序"
4. 切换回工作区 B，确认仍是"按修改时间排序"

### 错误场景测试
1. 在只读文件系统中测试（应显示错误消息）
2. 在没有工作区的情况下测试
3. 在空文件夹中测试

## 已知限制

1. **按大小排序**: VS Code API 不支持，使用文件优先模式替代
2. **按创建时间排序**: VS Code API 不支持，使用修改时间替代
3. **自定义排序**: 需要实现自定义树视图才能完全控制排序逻辑

## 改进建议

如果需要完整的排序功能，可以考虑：

1. 实现自定义的 TreeDataProvider
2. 手动读取文件系统信息（大小、创建时间等）
3. 完全控制文件树的显示顺序
4. 添加更多排序选项（如按扩展名、按路径深度等）

这需要更多的开发工作，但可以突破 VS Code 原生 API 的限制。
